# Machine Learning Essentials

This part of the course focused on the basics of how machine learning systems learn from data. The main idea is that machine learning allows a system to improve its behavior based on examples or patterns rather than relying only on manually written rules.

One of the most useful distinctions here was the difference between labeled and unlabeled data. Labeled data comes with known outputs, such as emails marked spam or not spam, or images labeled as cat or dog. Unlabeled data does not come with these answers in advance, so the model has to discover patterns or groupings on its own.

This naturally led into the three main learning paradigms:

## Supervised Learning
In supervised learning, the system is trained with labeled data. It learns a mapping between inputs and known outputs. This is used in tasks like classification and regression.

## Unsupervised Learning
In unsupervised learning, the system works with unlabeled data and tries to find hidden structure. This is useful for clustering, grouping, or discovering patterns that are not explicitly given.

## Reinforcement Learning
In reinforcement learning, an agent interacts with an environment and receives rewards or penalties. The model learns by trial and error to improve long-term outcomes.

I also learned the difference between an algorithm and a trained model. An algorithm is the method or process. A model is what we get after the algorithm has been trained on data. This sounds simple, but it is an important conceptual distinction.

Another useful idea was training versus inference. Training is the stage where the model learns. Inference is the stage where the trained model is used to make predictions or generate outputs.

Overall, this note helped me understand that machine learning is not one single technique. It is a family of approaches built around data, task structure, and learning strategy.
