# Learning Approach and Portfolio Direction

After going through the orientation, I decided to structure this repository around topic-wise learning blocks instead of lecture-wise storage.

That decision came from the idea that real learning happens in phases. A single topic often continues across multiple lectures, and understanding becomes clearer when related concepts, practice notebooks, and assignments stay together in the same place.

My learning approach for this course is based on four steps:

## 1. Study
I first try to understand the lecture material conceptually. This means paying attention to the big ideas, the definitions, and the relationships between topics instead of only focusing on code.

## 2. Practice
Whenever a notebook or practical demo is part of the lecture, I practice it in Google Colab and work through the code to understand how the theory becomes implementation.

## 3. Document
After studying and practicing, I write notes in my own words. This helps me move from passive recognition to active understanding. It also gives the repository more value because it reflects actual learning rather than copied material.

## 4. Build
Assignments are where the course moves from guided learning into proof of understanding. By placing assignments inside the topic they belong to, the repository shows a natural progression from theory to practice.

This repository therefore acts as:
- a study archive
- a documentation project
- a practical portfolio
- a personal progress tracker

The goal is not to make the repository look complete before it is complete. The goal is to make each completed part strong, clear, and useful.

That is why this portfolio is being built in a way that lets each topic stand on its own while still fitting into the larger AI learning journey.
