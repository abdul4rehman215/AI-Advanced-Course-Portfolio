# Strings, Lists, Tuples, Sets, and Dictionaries

This note covers the part of the Python section where the course focused on text handling and core collections.

## Strings
Strings represent text. I practiced:
- printing strings
- writing multiline strings
- converting text to uppercase or lowercase
- stripping spaces
- replacing text
- splitting text into parts

These methods made it clear that strings are not just static text. They can be processed, cleaned, transformed, and used in many practical ways.

## Lists
Lists are ordered and mutable collections. This means they keep item order and can be changed later. I practiced adding items, accessing elements by index, and slicing parts of the list.

## Tuples
Tuples are similar to lists, but they are immutable. This means once created, they are not meant to be changed. They are useful when fixed grouped values are needed.

## Sets
Sets are unordered collections that do not allow duplicates. This makes them useful when uniqueness matters more than position.

## Dictionaries
Dictionaries store key-value pairs. This is a powerful way to organize structured information such as names, settings, labels, or counts.

A major takeaway from this topic is that choosing the right data structure matters. Different collection types solve different problems. Learning these early helps build stronger logic in later notebooks and assignments.

This note also showed me that Python becomes much easier to use once the behavior of each collection type is clearly understood.
