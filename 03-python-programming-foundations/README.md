# 🧠 What I Studied In This Section

## 1) Programming mindset and ethics
The course did not begin Python by jumping directly into syntax. It first emphasized **how a programmer should think**.

Main ideas included:

- writing understandable code
- focusing on maintainability
- avoiding shortcuts that damage long-term quality
- building reusable logic
- treating code as something other people may later depend on

This was valuable because it framed programming as disciplined engineering rather than just “making something run.”

---

## 2) Variables and data types
I studied the basic building blocks of Python:

- integers
- floats
- strings
- variables as containers for values
- naming rules and good variable naming habits

This helped me understand how programs store information and how clear naming improves readability.

---

## 3) Strings and text manipulation
I practiced a range of string operations, including:

- printing strings
- multi-line strings
- `.upper()`
- `.lower()`
- `.strip()`
- `.replace()`
- `.split()`

This was useful because text handling becomes important in later AI tasks, APIs, prompts, and preprocessing.

---

## 4) Core data structures
I studied and practiced:

- lists
- tuples
- sets
- dictionaries

I learned how each data structure behaves differently in terms of:

- order
- mutability
- duplicates
- indexing
- key-value storage

This gave me a practical foundation for working with collections of data in Python.

---

## 5) Operators and logic building
I covered the main classes of operators used in Python:

- arithmetic operators
- comparison operators
- logical operators
- membership operators
- string concatenation and repetition

This part was important because operators are what allow programs to make logical decisions.

---

## 6) Conditional logic
I studied how control flow works through:

- `if`
- `if-else`
- `elif`

This taught me how to create branching program behavior based on conditions and input values.

---

## 7) Loops
I practiced both:

- `for` loops
- `while` loops

This helped me understand repetition, iteration, counter control, and condition-based execution.

I also saw how loops solve repetitive work much more effectively than copying the same code multiple times.

---

## 8) Functions
I studied functions as reusable units of logic.

Main ideas included:

- defining functions with `def`
- calling functions
- parameters
- return values
- local vs global scope
- avoiding repetition through reusable logic

This was a very important shift because functions are where programming starts becoming more structured and scalable.
