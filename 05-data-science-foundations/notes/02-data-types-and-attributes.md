# Data Types and Attributes

This note covers the different forms of data and the attributes used to describe it.

The course discussed the difference between:
- structured data
- unstructured data
- semi-structured data

## Structured Data
Structured data is organized in a clear tabular format, such as rows and columns. It is easier to query, filter, and analyze with traditional tools.

## Unstructured Data
Unstructured data includes forms like text, images, audio, and video. It does not naturally fit into clean tabular form and often requires more advanced methods to process.

## Semi-Structured Data
Semi-structured data sits somewhere in between. It may not fit into strict relational tables, but it still contains an internal organization, such as tagged or key-value formats.

The course also covered data attributes:
- nominal
- binary
- ordinal
- numeric

These attribute types matter because they influence:
- which visualizations make sense
- which preprocessing methods are needed
- which models can use the feature effectively

This topic helped me understand that not all data should be treated the same way. Good analysis depends on recognizing what kind of information is actually being handled.
