# Exploratory Data Analysis

EDA is one of the most important steps in data science because it helps us understand the data before modeling begins.

The course explained EDA as a way to inspect the dataset, spot patterns, identify unusual behavior, and decide what needs to be cleaned or transformed.

Three groups of ideas were emphasized:

## Central Tendency
These measures show where the data tends to concentrate.
- mean
- median
- mode

## Dispersion
These measures show how spread out the data is.
- variance
- standard deviation
- interquartile range

## Proximity
These measures show how close or related values are to each other.
Examples include correlation and similarity-related ideas.

I also learned that data distribution matters. A symmetric distribution behaves differently from a skewed one, and that changes how the data should be interpreted.

The main reason EDA matters is that it reveals the condition of the dataset before model training. If there are missing values, outliers, inconsistent columns, or weak features, EDA helps expose them early.

This note made it clear that EDA is not a small optional step. It is the part of the workflow that tells us what kind of data problem we are actually dealing with.
